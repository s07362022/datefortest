PUT/messages/{match_id}/readP4 標記對方訊息為已讀，回傳 marked_read 數量

🧪 Phase 4 測試指南

用 PowerShell 測試 WebSocket

# 1. 取得配對好的 Alice token + match_id（先確認有互 Like）
$ra = Invoke-RestMethod -Method Post -Uri "http://localhost:8000/auth/login" -ContentType "application/json" -Body '{"email":"alice@test.com","password":"Seed1234!"}'
$matches = Invoke-RestMethod -Uri "http://localhost:8000/matches" -Headers @{Authorization="Bearer $($ra.access_token)"}
$mid = $matches[0].id

# 2. 查看歷史訊息（REST）
Invoke-RestMethod -Uri "http://localhost:8000/messages/$mid" -Headers @{Authorization="Bearer $($ra.access_token)"} | ConvertTo-Json

# 3. 標記已讀
Invoke-RestMethod -Method Put -Uri "http://localhost:8000/messages/$mid/read" -Headers @{Authorization="Bearer $($ra.access_token)"} | ConvertTo-Json
# → {"marked_read": N}

# 4. WebSocket 測試需要 wscat / wscat npm 工具，或用瀏覽器 console：

📡 WebSocket 訊息協定

事件 (event)方向說明範例 Payload
connectedServer → Client連線建立確認{"event":"connected","match_id":1,"user_id":2}
發送訊息Client → Server傳送新訊息{"content":"Hello!"}
messageServer → All in room新訊息廣播{"event":"message","id":5,"sender_id":2,"content":"Hello!","created_at":"..."}
errorServer → Client錯誤通知{"event":"error","code":"UNAUTHORIZED","message":"Token 無效"}

錯誤碼列表

code說明
UNAUTHORIZEDToken 無效或未提供
MATCH_NOT_FOUND配對不存在或已取消
NOT_MATCH_MEMBER使用者不在此配對中
INVALID_JSON訊息格式非 JSON
EMPTY_CONTENT訊息為空
TOO_LONG訊息超過 2000 字
