Auth / Profiles / Preferences（Phase 1）

POST/auth/register註冊帳號

Photos / Tags / Discover（Phase 2）P2

POST/photos上傳照片（≤5MB, ≥400px）
GET/photos/me查看我的照片
DELETE/photos/{photo_id}刪除照片
PUT/photos/{photo_id}/primary設定主照片
GET/tags列出 30 個興趣標籤
GET/discover?limit=20推薦對象列表（需完整檔案）

Likes / Passes / Matches（Phase 3）P3 NEW

POST/likes/{user_id}喜歡某人；互相喜歡自動建立 Match，回傳 is_match: true
POST/passes/{user_id}略過某人（不再出現在 Discover）
GET/likes/quota今日 Like 使用量（50次/天，UTC 00:00 重置）
GET/matches查看所有配對成功的對象
DELETE/matches/{match_id}取消配對（雙向軟刪除）

💘 配對流程圖

互相喜歡 → 自動配對

A 按 POST /likes/B
→
Like 記錄寫入 DB
is_match: false
→
B 按 POST /likes/A
→
偵測到 A 已 Like B
→
建立 Match！
is_match: true

略過 → 永久排除

A 按 POST /passes/B
→
Pass 記錄寫入 DB
→
B 不再出現在 A 的 Discover

取消配對

DELETE /matches/{id}
→
is_active = false（軟刪除）
→
雙方的配對清單都消失

📋 業務規則

規則行為錯誤代碼
每日 Like 配額預設 50 次 / 天（UTC），超過回傳 HTTP 429DAILY_LIMIT_REACHED
自我 Like / Pass不允許對自己操作SELF_LIKE / SELF_PASS
重複 Like已操作過同一人則拒絕ALREADY_ACTED
重複 Pass靜默成功，冪等操作—
取消配對權限只有配對中的兩位用戶可以操作NOT_YOUR_MATCH
Match ID 唯一性以 min(uid1, uid2) 確保同一對只有一筆—
Discover 排除已 Like 或 Pass 的用戶不再顯示—

🧪 測試指南（Phase 3 配對功能）

【情境 A】Swagger UI 操作（最直觀）

開啟 http://localhost:8000/docs

用兩個 Seed 帳號分別登入並授權
帳號 1：alice@test.com / 帳號 2：bob@test.com，密碼皆 Seed1234!
（需要兩個瀏覽器分頁，各自 Authorize）
查 Bob 的 user_id — GET /auth/me（以 Bob token 登入查看）
Alice 喜歡 Bob — POST /likes/{bob_id}（以 Alice 授權）
回傳：{"is_match": false, "match_id": null, "message": "已送出喜歡"}
Bob 喜歡 Alice — POST /likes/{alice_id}（以 Bob 授權）
回傳：{"is_match": true, "match_id": 1, "message": "配對成功！"} 🎉
查看配對清單 — GET /matches（任一帳號）
可看到對方的 display_name 與主照片
查看今日 Like 配額 — GET /likes/quota
顯示 used: 1, limit: 50, remaining: 49
取消配對 — DELETE /matches/{match_id}
狀態碼 204，配對消失

【情境 B】PowerShell 一鍵測試腳本

# ── 登入兩個帳號 ──────────────────────────────
$ra = Invoke-RestMethod -Method Post -Uri "http://localhost:8000/auth/login" -ContentType "application/json" -Body '{"email":"alice@test.com","password":"Seed1234!"}'
$rb = Invoke-RestMethod -Method Post -Uri "http://localhost:8000/auth/login" -ContentType "application/json" -Body '{"email":"bob@test.com","password":"Seed1234!"}'

# ── 取得各自 ID ───────────────────────────────
$alice = Invoke-RestMethod -Uri "http://localhost:8000/auth/me" -Headers @{Authorization="Bearer $($ra.access_token)"}
$bob = Invoke-RestMethod -Uri "http://localhost:8000/auth/me" -Headers @{Authorization="Bearer $($rb.access_token)"}

# ── Alice 喜歡 Bob ────────────────────────────
Invoke-RestMethod -Method Post -Uri "http://localhost:8000/likes/$($bob.id)" -Headers @{Authorization="Bearer $($ra.access_token)"} | ConvertTo-Json
# → {"is_match": false, "match_id": null, "message": "已送出喜歡"}

# ── Bob 喜歡 Alice → 配對觸發！──────────────
$match = Invoke-RestMethod -Method Post -Uri "http://localhost:8000/likes/$($alice.id)" -Headers @{Authorization="Bearer $($rb.access_token)"}
Write-Host "配對成功？" $match.is_match # → True
Write-Host "Match ID ：" $match.match_id

# ── 查看 Alice 的配對清單 ─────────────────────
Invoke-RestMethod -Uri "http://localhost:8000/matches" -Headers @{Authorization="Bearer $($ra.access_token)"} | ConvertTo-Json

# ── 查看 Like 配額 ────────────────────────────
Invoke-RestMethod -Uri "http://localhost:8000/likes/quota" -Headers @{Authorization="Bearer $($ra.access_token)"} | ConvertTo-Json

【情境 C】略過測試

# Carol 略過 David → David 不再出現在 Carol 的 Discover
$rc = Invoke-RestMethod -Method Post -Uri "http://localhost:8000/auth/login" -ContentType "application/json" -Body '{"email":"carol@test.com","password":"Seed1234!"}'
$rd = Invoke-RestMethod -Method Post -Uri "http://localhost:8000/auth/login" -ContentType "application/json" -Body '{"email":"david@test.com","password":"Seed1234!"}'
$david = Invoke-RestMethod -Uri "http://localhost:8000/auth/me" -Headers @{Authorization="Bearer $($rd.access_token)"}
Invoke-RestMethod -Method Post -Uri "http://localhost:8000/passes/$($david.id)" -Headers @{Authorization="Bearer $($rc.access_token)"} | ConvertTo-Json
# → {"message": "已略過"}
